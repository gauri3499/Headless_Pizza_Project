promises:
      - it is an object ,used to handle asynchronomous operations

      States:
             1.pending ---starting /initial state 
             2.fulfilled -- if operation is sucess
             3.rejected -- if operation if failed

promises can be handle in two ways:

             1.then and catch 
                   then() -- a function,which is used to handler success of a promises
                   catch()-- a function ,which is used to deal with failure of a promises
                   finally() --a function , which executes even if we get success or failure

             2.async and await     
                 async is a function which hold a promises
                 await is a keyword used to wait for a promise to resolve
                 await is used inside async function(mandatory)

JSON :(javascript object notation)
======
      - it is a plain text, lightweight data interchange text
      - used to store data(noSQL) and exchange data(client<--->server)

                  JS Object                          JSON Object 
  --------------------------------------------------------------------------------------------------------
  syntax     flexible and dynamic                  fixed and static

  keyQoutes       optional                        mandatory

  JSON RULES:
  -----------
           - key must be indicated with double Qoutes
           - keys must be unique
           - Json object need to enclosed in {}
           - Json array need to enclosed in []

Json Types:
-----------
          1.json object
          2.json array
          3.Nested json
